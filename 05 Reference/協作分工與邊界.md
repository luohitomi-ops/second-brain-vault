# 協作分工與邊界（Gemini 外觀 × Claude 結構）

> 兩個 AI 分工維護這個 vault。劃清邊界，避免改壞會自動運作的部分。

## Gemini Pro 負責：視覺層（可自由改）
- `.obsidian/snippets/*.css` — 主題、顏色、字體、卡片光暈、背景
- `.obsidian/graph.json` 的顏色（colorGroups 的 color 值）
- Home 等頁面的**標題文字、emoji、排版順序、區塊樣式**
- 新增純展示用的圖片/裝飾

## Claude（我）負責：結構與邏輯層
- Markdown frontmatter（`type / area / status / updated`）— Dataview 靠這個
- ` ```dataview ` 程式碼區塊 — 改錯整個自動表格會壞
- `[[雙括號連結]]` — 神經網路靠這個
- 檔名（改名會斷所有連結）
- `.claude/commands/*` 自訂指令、各 `CLAUDE.md`
- 專案節點的增刪、資料更新

## ⚠️ 請 Gemini 不要動的東西
1. **任何 ` ```dataview ` 區塊內的查詢語法** — 只能改它外面的標題樣式
2. **frontmatter 欄位名與值** — 可以改不影響邏輯的，但別刪 type/area/status
3. **`[[...]]` 連結文字** — 改了會斷神經網路
4. **檔名** — 要改名告訴 Claude，由 Claude 連同所有連結一起改
5. **`.claude/` 與各 `CLAUDE.md`** — 那是大腦的神經系統，非視覺

## 衝突時
- 外觀 vs 邏輯衝突 → 邏輯優先（系統能運作 > 好看）
- 不確定某個檔能不能改 → 先問 Claude
