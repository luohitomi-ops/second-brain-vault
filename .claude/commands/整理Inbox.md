整理檸子的碎片靈感收件夾：

1. 讀 `07 Quick Capture & Hub/每日碎片化靈感隨記.md` 裡所有未處理的碎片。
2. 逐條判斷類型並處理：
   - **發文靈感** → 加 `#發文素材`，若需要搬去 [[每日熱門話題摘要]] 或留原地皆可（帶標籤即會進靈感候選庫）
   - **股票/投資知識** → 加 `#股票知識`，整理進 [[投資知識庫]]
   - **生活/醫美心得** → 整理進 [[愛美與醫美心得]]（帶 #發文素材 若可成內容）
   - **想學的技能** → 整理進 [[想學習的技能清單]]
   - **主持稿/直播稿**（某場活動實際拿到或自己準備的稿子、產品資料、注意事項）→ 整理成獨立檔案存進 `01 Brand/主持資料庫/01_事前準備/`，檔名格式 `直播腳本_案名_YYYYMMDD.md` 或 `主持稿_案名_YYYYMMDD.md`（照 [[直播腳本_滿貫大亨_20260630]] 的格式），完整保留原始稿子內容，不要精簡或摘要掉
   - **過往主持課程筆記**（上課學到的技巧、觀念，不是特定某場的稿子）→ 整理進 `01 Brand/主持資料庫/03_資料庫/`，格式照該資料夾 README 的範本（課程名稱/講師/日期 → 核心觀念 → 可套用技巧 → 實戰驗證）
   - **純雜念/待辦** → 轉成 `- [ ]` 待辦或刪除
3. 已處理的碎片，在 Inbox 該行前面標 `✅` 或移走（不要刪掉原始時間戳，保留軌跡）。
4. **重複候選檢查**（不自動刪，只提醒）：分流前，快速比對這條碎片的內容是否跟目標中樞（品牌中樞/COS經營/撥撥獺獺IP/投資知識庫等）裡已有的段落高度重疊。若疑似重複，不要默默整理掉，改成回報：「這條疑似跟 [[某中樞]] 的『某段落』重複，要我確認後清掉原始碎片嗎？」，等我回答才動手刪除或封存。
5. 回報：處理了幾條、各分到哪、有沒有無法判斷需要我確認的、有沒有偵測到重複候選。

原則：Inbox 只負責「3 秒捕獲」，分類由這個指令做，不要讓我手動整理。刪除/封存一律先問過我。
