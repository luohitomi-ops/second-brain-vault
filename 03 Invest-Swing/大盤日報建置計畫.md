# 大盤日報建置計畫（設計文件）

> 2026-07-18 brainstorming 產出，使用者已核准設計，待寫實作計畫。

## 背景 / 動機
使用者反映：目前股市每天變化極端，不適合每天糾結在個股細節數據判斷上。現有 `/財報`、`/建模`、`/研究` 都聚焦在**個股**，缺一個看「大方向」的每日總覽，且要求**真實依據，不能是 AI 亂掰數字**。

## 範圍
新建 `market-daily-report.mjs`，產生兩份每日報告，都推播到 #波段 + 存進 vault。

### 早上場（~07:30，開盤前功課）
- 美股三大指數收盤（道瓊/那斯達克/標普）
- 台指期夜盤情勢
- 前一交易日融資融券餘額增減（最新公布一筆）

### 下午場（~14:10，收盤後總結）
- 今日大盤指數(TAIEX) 漲跌
- 今日融資融券餘額增減
- 三大法人買賣超

### 手動指令
`/大盤`：隨時重新拉一次，不分早晚場，看當下能拿到的最新數據。

## 資料來源（已用真實請求驗證過）
| 項目 | 來源 | 驗證結果 |
|---|---|---|
| 大盤指數 | `https://www.twse.com.tw/rwd/zh/afterTrading/MI_INDEX?response=json&date=YYYYMMDD&type=IND` | ✅ 已測，回傳真實JSON |
| 融資融券餘額 | `https://www.twse.com.tw/rwd/zh/marginTrading/MI_MARGN?response=json&date=YYYYMMDD&selectType=ALL` | ✅ 已測 |
| 三大法人買賣超 | `https://www.twse.com.tw/rwd/zh/fund/BFI82U?response=json&date=YYYYMMDD` | ✅ 已測 |
| 美股三大指數 | `https://query1.finance.yahoo.com/v8/finance/chart/^GSPC`（`^DJI`/`^IXIC` 同法） | ✅ 已測，即時JSON |
| 台指期夜盤 | 期交所官網（`taifex.com.tw`），表單查詢格式非簡單GET JSON | ⚠️ 官方真實數據存在，但需要模擬表單POST，比其他四項工程複雜；第一版命中率待實測後誠實回報，抓不到就標「⚠️ 今日未能取得」不硬湊 |

全部免費、免API金鑰，跟現有 `holdings-price-updater.mjs`/`etf-monitor.mjs` 用同一套 TWSE OpenAPI 模式一致。

## 誠實原則（核心需求，不可妥協）
- 抓不到的數據項目一律標「⚠️ 今日未能取得，數據來源可能異常」，絕不用AI編造補齊
- AI（Gemini flash-lite，0 Claude額度）只負責把「這次真實抓到的數字」組成 2-3 句情境描述（例如「外資賣超擴大、融資餘額續減，籌碼面轉保守」），**明確禁止引用未提供的數據或臆測未來走勢**
- 呼應既有 standing rule（`feedback_investment_advisory_tone`）：波段類給情境分析，不下指令，不說「該買該賣」
- 每則報告附數據來源時間戳，方便事後核對是不是真的當天數據

## 儲存 + 排程整合
- 累積寫進 `03 Invest-Swing/大盤日報.md`（仿照 `財報摘要.md` 按日期累加的做法）
- discord-bot.mjs 新增兩個排程時間點：平日 07:30、平日 14:10（跟現有 `/槓桿` 14:05 錯開，避免同時發送請求撞在一起）
- 新增 Discord 文字指令 `/大盤`（沿用現有 `/財報`、`/建模` 的模式：偵測文字→呼叫函式→回報+推播）

## 不在這次範圍內
- 不做個股層級分析（已有 `/財報`、`/建模`、`/研究` 涵蓋）
- 不做預測/建議買賣（違反既有波段顧問態度 standing rule）
- 夜盤數據源如果第一版工程量超出預期，可以先上線其他四項，夜盤部分列為後續迭代項目，不強制一次到位
