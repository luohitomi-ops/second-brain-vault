# 給獺金幣程式（Stock Radar）

狀態：beta｜類型：台股族群 Gap 篩選工具 / 自用
- 線上：https://stock-radar2.vercel.app
- Repo：https://github.com/luohitomi-ops/stock-radar

## 核心邏輯
- **Gap =（驅動指數報酬 × Beta）−台股族群均報酬**；正 Gap=台股落後有補漲機會
- 驅動指數：SOX(Beta 0.8)、SPX(0.6)、N225(0.7)
- VIX 風控：<20 正常｜20-25 減碼｜≥25 避免進場
- 最小篩選：Gap 1.5pp、Beta ≥ 0.5

## 排程（台灣時間）
- **主用：Windows 工作排程器**（電腦24h開，比雲端可靠）：
  - `StockRadar_Notify` 平日 08:33（TG 推播）
  - `StockRadar_Snapshot` 平日 16:03（快照存 Redis）
  - 腳本：`C:\tia\stock_radar_notify.ps1`、`stock_radar_snapshot.ps1`
- **備援：Vercel cron**（08:30/16:00，Hobby 方案不穩故僅備援；偶爾兩邊都跑會收到重複 TG，可接受）
- ⚠️ 重灌/換機要重建這兩個本機排程

## API 端點
- `/api/health` — 快照新鮮度（回 fresh/yesterday/stale）← 大腦健檢就打這支
- `/api/snapshot`、`/api/notify`(TG)、`/api/ai`、`/api/sync`、`/api/price-history`

## 大腦健檢整合
- 自動：平日 08:45 bot 自動健檢 → 貼 #總控
- 手動：Discord 任一頻道打 `/健檢`
- 檢查項：快照是否今日更新；TG 是否準時（沒收到=本機排程沒打 /api/notify）
