# 主持資料庫

> 主持/直播工作的統一管理節點。原「主持與直播執行庫」單檔升級為資料夾。

## 結構

| 子項目 | 用途 |
|---|---|
| [[01_事前準備/README\|01_事前準備 資料夾]] | 主持前拿到的稿子、產品資料、注意事項、自做功課 → 潤稿/備忘錄；2026-07-18 起自動依檔名日期分「即將開始」/「過期待歸檔」兩區 |
| [[主持知識備忘]] | 萬用主持知識庫，從過期主持稿蒸餾出的通用內容（開場話術/突發狀況處理等），按主題分類非按場次 |
| [[02_主持復盤]] | 隨筆式當日狀況與優缺點復盤，不用制式表格 |
| [[03_資料庫/README\|03_資料庫 資料夾]] | 過去上過的主持課程筆記，整理歸納後的統一版本（待填） |
| [[主持與直播執行庫]] | 場次紀錄範本 + 長期品牌日誌（原檔案內容，保留原檔名） |

## 定期清理原則（2026-07-18 更新，取代舊版「移到05 Reference」規則）
- **會過期的資料**：單場活動的臨時稿子/直播腳本，`01_事前準備/README.md` 開啟時會自動偵測檔名日期已過去的、跳出提醒。跟使用者確認後把裡面還有用的內容蒸餾進 [[主持知識備忘]]（按主題分類），原稿確認蒸餾完再刪除——不是搬去 `05 Reference` 封存，是萃取完直接刪，避免同樣內容留兩份
- **核心資料**：報價邏輯、長期合作品牌日誌、已蒸餾好的課程筆記 → 留在此資料夾分類保存，不清理
