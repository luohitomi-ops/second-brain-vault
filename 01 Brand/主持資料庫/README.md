# 主持資料庫

> 主持/直播工作的統一管理節點。原「主持與直播執行庫」單檔升級為資料夾。

## 結構

| 子項目 | 用途 |
|---|---|
| [[01_事前準備/README\|01_事前準備 資料夾]] | 主持前拿到的稿子、產品資料、注意事項、自做功課 → 潤稿/備忘錄；2026-07-18 起自動依檔名日期分「即將開始」/「過期待歸檔」兩區 |
| [[主持知識備忘]] | 萬用主持知識庫，從過期主持稿蒸餾出的通用內容（開場話術/突發狀況處理等），按主題分類非按場次 |
| [[02_主持復盤]] | 隨筆式當日狀況與優缺點復盤，不用制式表格 |
| [[03_資料庫/README\|03_資料庫 資料夾]] | 過去上過的主持課程筆記，整理歸納後的統一版本（待填） |
| [[主持與直播執行庫]] | 場次紀錄範本 + 長期品牌日誌（原檔案內容，保留原檔名） |

## 定期清理原則（2026-07-20 修正：區分單次活動 vs 長期合作客戶）
> ⚠️ **2026-07-20 踩過的坑**：把「滿貫大亨」直播腳本（含整場時間軸/送禮節奏/口播細節）蒸餾進 [[主持知識備忘]] 後刪除原稿，結果使用者要規劃下一場滿貫時發現原稿不見了——蒸餾出來的是「通用結構模板」，不是「這個客戶下次還能直接照抄的具體操作稿」，兩者不能互相取代。已用 vault git 備份救回，並修正規則如下：

- **長期合作客戶**（有出現在下方 [[主持與直播執行庫]]「長期品牌日誌」表的客戶，例如滿貫大亨/傑太ploom）：腳本/速查表是**下次同客戶還會直接複用/微調的操作範本**，過期後**不刪**，改移到 `03_資料庫/` 歸類保存（跟課程筆記同一層級，都是「以後還要拿出來用」的東西）；[[主持知識備忘]] 只負責蒸餾「跨客戶都通用」的抽象原則，兩份並存不衝突
- **單次活動**（不在長期品牌日誌表裡的一次性場次，例如KYMCO重機展這種）：照舊，過期後蒸餾進 [[主持知識備忘]]、原稿確認蒸餾完才刪除
- **核心資料**：報價邏輯、長期合作品牌日誌、已蒸餾好的課程筆記 → 留在此資料夾分類保存，不清理
