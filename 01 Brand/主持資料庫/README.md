# 主持資料庫

> 主持/直播工作的統一管理節點。原「主持與直播執行庫」單檔升級為資料夾。

## 結構

| 子項目 | 用途 |
|---|---|
| `01_事前準備/` 資料夾 | 主持前拿到的稿子、產品資料、注意事項、自做功課 → 潤稿/備忘錄 |
| [[02_主持復盤]] | 隨筆式當日狀況與優缺點復盤，不用制式表格 |
| [[03_資料庫/README\|03_資料庫]] 資料夾 | 過去上過的主持課程筆記，整理歸納後的統一版本（待填） |
| [[主持與直播執行庫]] | 場次紀錄範本 + 長期品牌日誌（原檔案內容，保留原檔名） |

### 📂 01_事前準備 目前檔案（自動列出，新增檔案會自動出現）

```dataviewjs
const files = dv.pages('"01 Brand/主持資料庫/01_事前準備"').sort(p => p.file.mtime, 'desc').array();
if (files.length === 0) {
  dv.paragraph("目前沒有檔案。");
} else {
  dv.table(["檔案", "類型", "建立日期"], files.map(f => [
    f.file.link,
    f.type || "-",
    f.created || f.file.cday.toFormat("yyyy-MM-dd")
  ]));
}
```

## 定期清理原則
- **會過期的資料**：單場活動的臨時稿子、已結束場次的舊備忘錄 → 定期移到 `05 Reference` 封存，不留在此資料夾造成雜訊
- **核心資料**：報價邏輯、長期合作品牌日誌、已蒸餾好的課程筆記 → 留在此資料夾分類保存，不清理
