## content_plan.md/history.md 資料落差根因排查（2026-07-12～07-15）
分好幾天陸續發生「DC bot 端跟桌面端資料對不上」的狀況，逐一排查後確認是多個獨立小問題疊加，非單一根因：
- **7/12**：`backfill-parser.mjs` 認不出「分」縮寫（只認完整「分享」）、`extractFormula()` 沒砍時間 HH:MM 導致標題欄 garbled——兩者皆已修。
- **7/12**：Obsidian Git 外掛 `autoPullInterval:15`（每15分鐘自動pull）跟 Discord bot／Home.md 儀表板兩個獨立寫入來源理論上有碰撞風險——已拉長到60分鐘降低機率（使用者一天只更新一次社群數據，可接受）。
- **7/13**：`/雙週復盤` 產生新一輪日曆時沒清舊一輪還沒發生的重疊尾巴，造成同日期兩輪並存——已修指令規則，新一輪覆蓋舊一輪日期時強制清尾巴。
- **7/15**：真正的路徑根因——`.claude/commands/發文日常.md`、`爆款優化.md` 寫死 `C:\Users\USER\.claude\skills\social-post\` 底下的 symlink 路徑，Edit 工具拒絕透過 symlink 寫入（工具內建安全機制，非權限問題），導致 DC 端 session 一度誤以為「有兩份 content_plan.md 各自被獨立自動化維護」——經 hash 比對證實只有一份檔案（symlink 真身在 vault），是 session 自己的錯誤推論。已統一改用 vault 真實路徑，DC 端與桌面端共用同一份指令檔，一次修好兩邊。
- 路徑規則詳見 [[social-post更新規則]]「寫入💎數據檔的路徑規則」。
