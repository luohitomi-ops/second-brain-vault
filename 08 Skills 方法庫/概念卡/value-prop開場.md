# value-prop-lead 開場

**核心**：開頭先講「這幫你解決什麼／得到什麼」，不要用 meta 故事或懸念式好奇當開場。讀者要的是「這跟我有什麼關係」，不是「猜猜我要說什麼」。

## 來源
- R37 規則（`references/rules.md`）：同一個工具兩種開頭，value-prop 開場比 meta 故事版本儲存數多 21 倍（466 vs 22 存）

## 跟 demo>claim 的差異
- [[demo大於claim]] 是「成品 vs 宣稱」的對比
- 這張卡是「開頭講用途 vs 開頭講故事」的對比，是同一母規律在「開場句」層的具體展現

## 跨領域應用
- **社群貼文**：開頭第一句就講清楚「這篇/這個東西幫你做什麼」
- **部落格長文**：標題+開場 100-150 字必須是 value prop，不是懸念式鉤子（見 `/部落格寫作` 指令）
- **業配提案 / 報價**：跟客戶說明服務時，先講「這解決你什麼問題」，不是先講品牌故事
